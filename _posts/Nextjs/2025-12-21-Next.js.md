ここではNext.jsについて学習します。  

## 以前の方法（Pages Router）との違い  
| 項目 | 以前の方法（Pages Router） | 新しい方法（App Router） |
|---|---|---|
| 使うフォルダ名 | `pages` | `app` |
| URLの決め方 | ファイル名がURLになる | フォルダ名がURLになる |
| 例（`/about` ページ） | `pages/about.tsx` というファイルを作る | `app/about/` というフォルダを作る |

create-next-appで新しいプロジェクトを作るときに、「Would you like to use App Router?」と質問されるのは、  
このどちらの方法でウェブサイトを構築するかを選んでいる、ということなのです。  

export default function Page() { ... }: この構文は、このファイルがWebページであることをNext.jsに伝えるための決まり文句です。  

## importする際のクラスを参照する方法は2通りあります。  
1. ドット表記  object.property  
クラス名がケバブケース（ハイフン繋ぎの記法のこと。例えば、class-nameなど）でない場合にはドット表記を使うことができます。  

2. ブラケット表記（[]）  object["property"]  
クラス名がケバブケースの場合にはブラケット記法を使うことができます。  

| 書き方                  | 意味       |
| -------------------- | -------- |
| `./page.module.css`  | 同じフォルダ   |
| `../page.module.css` | 1つ上のフォルダ |
| 違い                   | パス（場所）だけ |

## Linkコンポーネントをnext/linkからインポートして使用します  
主に以下のpropsを指定します。  

- href  
pagesフォルダまたはappフォルダ内のパスを指定する  
必須となる  
-  replace  
初期値はfalseであり真偽値を指定する  
trueの場合は、指定されたURLに遷移する際に、現在のブラウザの履歴に新しいURLを追加する代わりに書き換える  
- scroll  
初期値はtrueであり真偽値を指定する  
trueの場合は、指定されたURLに遷移した後にページトップにスクロールする  
-  prefetch  
初期値はtrueであり真偽値を指定する  
trueの場合は、指定されたURLのページをプリフェッチする
